# 量化研究规划 v2（整合开源工具链版）

> **AI工程师视角 × 股票量化 × 供应链传导** 你的独特优势：懂AI技术 + 硬件供应链认知。量化模型的目标是把这种认知优势转化为可验证的交易信号。
> 
> **v2 核心变化**：不再自建轮子。以 **Qlib 为研究底座**、**RD-Agent 为因子挖掘加速器**、**QMT/xtquant 为实盘通道**，你的精力集中在"供应链传导因子"这个别人没有的 alpha 上，基础设施全部复用开源成熟方案。预计整体时间线从 33 周压缩到 **约 24 周**。

## 一、认知地图：8个领域 × 3条供应链（保留 v1）

### 1.1 供应链传导图

```
AI算力需求 ──→ 芯片(GPU/CPU/ASIC) ──→ 显卡/GPU ──→ 电脑设备/数据中心
                     │                      │
                     └─→ PCB(封装基板) ──→ ──┘
                                           │
光纤 ──→ 数据中心 ←── 数据库(数据基础设施)    └─→ 工业设备(制造检测)
```

### 1.2 三条核心传导链

|传导链|路径|传导逻辑|量化信号|
|---|---|---|---|
|**算力链**|AI需求 → 芯片 → 显卡 → 数据中心/电脑|AI训练需求增长 → GPU订单增加 → PCB/封装需求上升|NVIDIA财报 → 国内GPU公司营收 → PCB龙头订单|
|**基础设施链**|数据需求 → 数据库 → 光纤 → 数据中心|数据量增长 → 存储需求 → 通信带宽需求 → 服务器部署|云厂商资本开支 → 光模块出货量 → IDC租金|
|**制造链**|芯片/显卡需求 → 工业设备 → PCB → 电脑设备|芯片产量增加 → 检测/制造设备需求 → PCB产能扩张 → 终端出货|联想出货量 → PCB龙头产能利用率 → 检测设备订单|

### 1.3 家庭信息渠道 → ⚠️ 合规边界（v2 强化）

**只能用公开信息，这条线必须画死。**

|信息类型|能否使用|说明|
|---|---|---|
|联想季度出货量（IDC/Canalys/财报公布后）|✅ 可用|公开数据，任何人可得，属合法领先指标研究|
|行业层面的常识性判断（"今年PC需求整体偏弱"）|✅ 可用|属于行业认知，非具体未公开事实|
|某供应商出现质量问题、即将被替换|❌ **不可用**|这是关于该供应商（可能是A股上市公司）的**未公开重大信息**，据此交易可能构成内幕交易|
|某供应商新获联想订单（未公告）|❌ **不可用**|同上，订单转移信息在公告前属内幕信息|
|内部BOM成本、未公开的采购价格|❌ **不可用**|未公开商业信息|

> v1 里"质量问题追踪 / 供应链切换 / BOM成本变化"三项作为量化信号的想法**删除**。父亲的价值在于帮你建立**行业理解和验证公开数据的解读**（比如教你看懂产能利用率、良率、供应商分级体系意味着什么），而不是提供具体的未公开事实。这不仅是法律风险，也是他的职业风险。

### Phase 2：因子研究 — ⏱ 5-6 周（v1: 6-8周）

**双轨并行：**

**轨道 A：站在 Alpha158 肩膀上（第1-2周）**

- 直接跑 Qlib 内置 Alpha158/Alpha360 因子库，用其 IC/ICIR/Rank IC 评价体系建立基准
- 你的自定义因子（基本面/技术面）用 Qlib 表达式引擎实现，与 Alpha158 同框比较——只有**相对基准有增量 IC** 的因子才保留

**轨道 B：供应链传导因子（你的核心 alpha，第2-6周）**

|传导假设|上游指标|下游标的|验证方法|
|---|---|---|---|
|AI需求 → GPU公司|NVIDIA营收/数据中心业务增速|海光信息/寒武纪|Granger因果 + 滚动回归|
|GPU需求 → PCB|GPU出货量|深南电路/沪电股份|领先1-2个月的交叉相关|
|云资本开支 → 光模块|AWS/阿里云/腾讯云资本开支|中际旭创/新易盛|领先3-6个月传导|
|PC出货 → 供应商|IDC/Canalys 公开出货量|鹏鼎控股/深南电路|公开数据 + 行业理解辅助解读|

供应链因子在 Qlib 里作为**自定义特征列**注入（低频月度数据 forward-fill 对齐日频，注意用发布日期而非报告期对齐，防前视偏差）。

**轨道 C：RD-Agent 自动挖掘（并行实验，不占主线时间）**

- 部署 `microsoft/RD-Agent`，跑 `fin_factor` 场景：LLM 自动"假设生成→代码实现→Qlib回测→反馈迭代"
- 后端可尝试接你本地部署的 Qwen3（OpenAI 兼容接口），跑不动再切 API
- ⚠️ 已知风险（券商实测结论）：LLM 偶尔臆造数据字段、忽视前视偏差，产出因子**必须人工审查后**才能进因子库——正好套用你熟悉的"Claude 审核门控"工作模式
- 注意 RD-Agent 与 Qlib 的 Python 版本兼容问题，建议独立 conda 环境

### Phase 3：模型构建 — ⏱ 6-8 周（v1: 8-12周）

|子模块|v2 实现方式|产出|
|---|---|---|
|**多因子选股**|Qlib LightGBM baseline（周/月频调仓，TopkDropout 策略）→ 有效后再试 QuantML-Qlib 的 iTransformer/PatchTST 等|每板块 Top 3 个股|
|**板块轮动**|相对强弱 RS + 宏观因子（自研，Qlib 外挂模块）|月度板块配置权重|
|**供应链因果**|VAR/Granger + 滚动回归（自研核心）|跨板块交易信号|
|**NLP 情绪**|本地 Qwen3 批量处理财报/公告/研报 → 情绪得分作为 Qlib 特征列|情绪因子|

**顺序纪律**：LightGBM 基线没跑赢简单指数增强之前，不碰深度模型。树模型在表格化因子数据上依然是最难打败的基线。

**NLP 情绪模型的落地路径**（你的第二 alpha）：

1. 东方财富/巨潮公告爬取 → 本地 Qwen3 结构化抽取（业绩预告方向、订单公告金额、管理层措辞变化）
2. 输出标准化情绪分（-1 ~ +1）+ 事件类型标签
3. 事件研究法验证：公告后 1/5/20 日超额收益分布
4. 有效事件类型进入 Qlib 特征

### Phase 4：回测与验证 — ⏱ 3-4 周（v1: 4-6周）

Qlib 自带回测框架，你要做的是**把成本参数调到 A 股现实水平**（可参考 OSkhQuant 的费用建模颗粒度）：

|参数|设置|说明|
|---|---|---|
|佣金|万2.5，最低5元/笔|按你实际券商费率调整|
|印花税|卖出单向 0.05%|现行税率|
|滑点|0.1%-0.2% 或按成交量占比建模|小市值标的调高|
|涨跌停|涨停不可买入、跌停不可卖出|Qlib 回测需显式配置 limit_threshold|
|T+1|当日买入不可卖出|必须启用|

**回测铁律（沿用 v1，全部保留）**：

|铁律|说明|
|---|---|
|幸存者偏差|股票池含已退市股（Qlib 官方 A 股数据集已处理，自建数据需注意）|
|前视偏差|财报因子用**公告日**对齐；Qlib Point-in-Time 数据库专治此问题|
|样本外验证|2020-2024 训练 → 2025-2026 验证，硬要求|
|滚动窗口|因子权重/模型每季度重训练（Qlib 支持 rolling retrain 工作流）|
|模型衰减监控|每月计算因子 IC 的滚动均值，连续衰减触发预警|

## 六、核心原则（v2 修订）

1. **不重复造轮子**：基础设施用 Qlib，精力全部投入供应链因子和 NLP 情绪这两个自有 alpha
2. **数据不编造**：没查到的标注 [MISSING]；财报因子严格按公告日对齐
3. **基线纪律**：LightGBM + Alpha158 是所有模型的及格线，跑不赢基线的复杂模型一律不上
4. **合规红线**：只用公开信息；父亲提供行业理解而非未公开事实——这条保护的是你们两个人
5. **人工门控**：RD-Agent 产出因子、模型交易信号，实盘前3个月全部人工审核后执行
6. **风控高于收益**：8个领域本质是一个高相关的科技板块，注意组合层面的集中度风险
