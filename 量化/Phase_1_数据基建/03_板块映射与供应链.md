## 一、板块分类标准

每个标的按以下维度分类：

| 维度 | 说明 | 示例 |
|------|------|------|
| **主领域** | 核心业务所属领域 | 深南电路 → PCB |
| **子领域** | 领域内的细分 | 高频高速PCB / IC载板 |
| **供应链位置** | 上游/中游/下游 | PCB → 中游 |
| **跨领域映射** | 同时涉及多个领域 | 深南电路 = PCB + 封装基板 |

## 三、供应链关系图谱（数据结构）

用SQLite存储传导关系：

```sql
-- 标的映射表
CREATE TABLE stocks (
    ts_code TEXT PRIMARY KEY,  -- Tushare代码
    name TEXT,
    sector TEXT,               -- 8个领域之一
    sub_sector TEXT,
    supply_chain_pos TEXT,     -- upstream/midstream/downstream
    cross_sector TEXT          -- 跨领域映射
);

-- 供应链传导关系表
CREATE TABLE supply_chain_links (
    id INTEGER PRIMARY KEY,
    upstream_code TEXT,        -- 上游标的代码
    downstream_code TEXT,      -- 下游标的代码
    lag_months INTEGER,        -- 预计传导时效（月）
    strength FLOAT,            -- 传导强度假设（0-1）
    verified BOOLEAN,          -- 是否已用数据验证
    last_verify_date TEXT      -- 最近验证日期
);
```
