# 因子库设计

> 因子是量化研究的原子。定义清楚，后面的工作才有方向。

---

## 一、因子分类体系

```
因子库
├── 基本面因子（Fundamental）
│   ├── 估值因子：PE/PB/PS/EV/EBITDA
│   ├── 成长因子：营收增速/利润增速/研发增速
│   ├── 质量因子：ROE/毛利率/研发占比/应收周转
│   └── 分红因子：股息率/分红率/分红连续性
│
├── 技术面因子（Technical）
│   ├── 动量因子：过去N日收益率
│   ├── 反转因子：过去N日收益率（反向）
│   ├── 波动因子：历史波动率/ATR
│   ├── 量价因子：换手率/量比/成交额变化
│   └── 流动因子：Amihud非流动性/买卖价差
│
├── 供应链因子（Supply Chain） ← 你的独特alpha
│   ├── 上游传导因子：上游出货量→下游营收预测
│   ├── 产能因子：产能利用率/产能扩张计划
│   ├── 库存因子：库存周转天数变化
│   └── 订单因子：新订单增速/订单交付周期
│
└── 事件因子（Event-driven）
    ├── 政策因子：信创/AI政策事件收益率
    ├── 财报因子：业绩超预期/低于预期
    ├── 产品因子：新产品发布事件收益率
    └── 供应链事件：供应商切换/质量问题事件
```

## 三、因子检验方法

### IC（信息系数）检验

IC = 因子值与未来N日收益率的**秩相关系数**（Spearman）。

| IC范围 | 解释 | 行动 |
|--------|------|------|
| |IC| > 0.05 | 有效因子 | 纳入模型 |
| |IC| > 0.03 | 弱有效 | 观察衰减 |
| |IC| < 0.03 | 无效 | 暂不使用 |
| IC波动大 | 不稳定 | 需要正交化/平滑 |

### Granger因果检验（供应链因子专用）

验证"上游指标变化是否**领先**下游股价变化"：

```python
from statsmodels.tsa.stattools import grangercausalitytests

# 假设：NVIDIA营收增速 Granger-因果 海光信息股价
result = grangercausalitytests(
    data[['nvidia_revenue_growth', 'hygon_stock_return']],
    maxlag=4  # 检验1-4个月领先
)
```

| Granger结果 | 解释 | 量化用法 |
|-------------|------|---------|
| lag=1显著 | 上游领先1个月 | 月度调仓用 |
| lag=2显著 | 上游领先2个月 | 中期配置用 |
| 不显著 | 无因果关系 | 不纳入传导模型 |

## 四、因子组合原则

| 原则 | 说明 | 原因 |
|------|------|------|
| **因子正交化** | 用PCA/排名正交化去除因子间相关性 | IC高但重叠的因子=虚假alpha |
| **IC加权** | 因子权重 = 近12个月IC均值/IC标准差 | IC越高→权重越大 |
| **因子衰减检测** | 每季度重新计算IC | 因子可能失效 |
| **领域差异化** | 不同领域用不同的因子组合 | PCB看毛利率+产能；芯片看研发+增速 |

### 各领域因子组合建议（初版）

| 领域 | 核心基本面因子 | 核心技术面因子 | 核心供应链因子 |

| 芯片 | 研发占比 + 营收增速 | 60日动量 | 晶圆产能利用率 |
| PCB | 毛利率 + 库存周转 | 20日反转 | PCB产能利用率 + 下游出货量 |
| GPU | 营收增速 + 毛利率 | 20日动量 | NVIDIA营收增速 + 云CapEx |
| 光模块 | 营收增速 + 海外占比 | 60日动量 | 云CapEx + 400G渗透率 |

| 电脑设备 | 出货量增速 + BOM占比 | 20日动量 | 联想出货量 |
