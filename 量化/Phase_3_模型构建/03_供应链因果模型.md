## 一、供应链传导的核心逻辑

### 1.1 三条传导链详解

#### 算力链：AI需求 → GPU → 光模块 → PCB

```
AI训练/推理需求增长
    │
    ↓ (需求传导, 0-1月)
GPU订单增加 (NVIDIA/AMD/海光)
    │
    ↓ (生产传导, 1-2月)
PCB订单增加 (深南电路/鹏鼎控股)
    │
    ↓ (组装传导, 2-3月)
服务器出货量增加 (浪潮/曙光)
    │
    ↓ (部署传导, 1-3月)
光模块需求增加 (中际旭创/新易盛)
    │
    ↓ (基础设施传导, 3-6月)
数据中心扩容 → 光纤铺设 (长飞光纤)
```

**量化验证方法**：
- NVIDIA季报营收增速 → 下月国内GPU公司股价相关性
- 云厂商CapEx季度数据 → 下季度光模块公司营收增速
- 滚动回归：上游X%增长 → 下游Y%增长的可预测性

#### 基础设施链：数据 → 存储 → 传输 → 计算

```
数据量增长 (AI训练数据/企业数据)
    │
    ↓ (1-3月)
数据库需求增长 (达梦/星环)
    │
    ↓ (3-6月)
存储硬件需求 (服务器+SSD)
    │
    ↓ (6-12月)
通信带宽需求 (光纤+光模块)
```

#### 制造链：需求 → 设备 → PCB → 终端

```
终端需求 (PC/服务器出货)
    │
    ↓ (1-2月)
元器件需求 (芯片/PCB/连接器)
    │
    ↓ (2-4月)
制造设备需求 (激光/检测/自动化)
    │
    ↓ (4-8月)
产能扩张 → 下游出货量增加
```

## 三、供应链因果模型的技术实现

### 3.1 Granger因果检验

```python
import pandas as pd
from statsmodels.tsa.stattools import grangercausalitytests

def test_supply_chain_causality(upstream_series, downstream_series, max_lag=6):
    """
    测试上游指标是否Granger-因果下游股价/营收
    max_lag: 最大检验的滞后期（月数）
    """
    data = pd.DataFrame({
        'upstream': upstream_series,
        'downstream': downstream_series
    }).dropna()

    results = grangercausalitytests(data, maxlag=max_lag, verbose=True)

    # 提取显著的滞后期
    significant_lags = []
    for lag in range(1, max_lag + 1):
        f_test_p = results[lag][0]['ssr_ftest'][1]
        if f_test_p < 0.05:
            significant_lags.append(lag)

    return significant_lags  # 返回显著领先的月数
```

### 3.2 滚动回归（传导强度估计）

```python
import numpy as np
from sklearn.linear_model import LinearRegression

def rolling_regression(upstream, downstream, window=12, lag=2):
    """
    滚动回归：估计上游对下游的传导系数
    window: 滚动窗口（月数）
    lag: 上游领先下游的月数
    """
    coefficients = []

    for i in range(window, len(upstream) - lag):
        X = upstream.iloc[i-window:i].values.reshape(-1, 1)
        y = downstream.iloc[i-window+lag:i+lag].values

        model = LinearRegression().fit(X, y)
        coefficients.append(model.coef_[0])

    return pd.Series(coefficients,
                     index=downstream.index[window:len(downstream)-lag])
```

### 3.3 VAR模型（多变量传导）

```python
from statsmodels.tsa.api import VAR

def var_supply_chain(data_dict, max_lags=4):
    """
    VAR模型：多变量供应链传导分析
    data_dict: {'nvidia_rev': series, 'hygon_price': series, ...}
    """
    data = pd.DataFrame(data_dict).dropna()
    model = VAR(data)
    results = model.fit(maxlags=max_lags, ic='aic')

    # 脉冲响应：NVIDIA营收冲击 → 海光信息股价响应
    irf = results.irf(10)
    irf.plot()  # 可视化传导路径

    return results
```

## 四、你父亲的信息如何合规使用

### 4.1 可以用的（公开或经授权的趋势信息）

| 信息类型 | 举例 | 量化用途 |
|----------|------|---------|
| PC出货量趋势 | "联想本季度PC出货量同比增长约X%" | 终端需求领先指标 |
| 供应商质量问题频率 | "最近XX供应商质量问题增多"（不指名） | 行业景气度的反面信号 |
| 行业采购趋势 | "下半年服务器采购量预计增长" | 服务器/算力需求预判 |
| BOM成本趋势 | "芯片成本在涨/跌" | 成本传导预判 |

### 4.2 不能用的（内幕信息）

| 信息类型 | 举例 | 原因 |
|----------|------|------|
| 未公布的财务数据 | 联想下季度具体利润数字 | 内幕信息，违法 |
| 具体供应商切换 | "联想决定从A供应商切换到B供应商" | 影响A/B公司股价的未公开信息 |
| 客户订单细节 | "某客户下了XX万台的订单" | 未公开的商业信息 |

> 💡 **合规底线**：使用的信息必须是"一个合理的投资者能通过公开渠道获取的"同等级别信息。你父亲提供的**趋势方向**（如"出货量在增长"）通常是可以的，但**具体数字**和**具体公司名称**可能涉及内幕。建议和父亲确认边界。

## 五、供应链模型的更新机制

| 更新频率 | 内容 | 方法 |
|----------|------|------|
| 每季度 | 上游财报数据更新 → 重算传导系数 | 滚动回归重新拟合 |
| 每半年 | Granger因果重新检验 → 时效是否变化 | 因果检验重跑 |
| 每年 | 供应链结构是否变化（新进入者/退出者） | 人工审查 + 标的映射更新 |
| 事件驱动 | 重大供应链事件（制裁/断供/新技术） | 人工判断是否需要调整模型 |
